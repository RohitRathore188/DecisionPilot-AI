1. Problem Statement

1.1 The Core Business Challenge

Small and Medium Enterprises (SMEs) — particularly in the food service, retail, and catering industries — face a critical and recurring challenge: making high-stakes business decisions without access to data-driven intelligence tools.

Traditional decision-making in SMEs relies on:

  • Gut feelings and past experience, without quantitative backing
  • Manual spreadsheet analysis that is error-prone and time-consuming
  • Expensive external consultants who are not accessible 24/7
  • Siloed business data — inventory, suppliers, financials — that is never correlated
  • Reactive rather than proactive operational strategies

1.2 Specific Pain Points Addressed

Pain PointImpact on BusinessDecisionPilot AI Solution
No scenario comparison tools Decisions made on single-point estimates; no understanding of risk Multi-scenario Monte Carlo simulation with ROI comparison
Inventory mismanagement Stockouts or overstock leading to waste and lost revenue Real-time stock level monitoring with AI restock predictions
Supplier dependency risk Single-supplier reliance creates fragile supply chains Supplier reliability scoring and multi-supplier comparison
No financial forecasting Operating blind with no projection of revenues and costs Multi-scenario revenue, cost, and profit projections
Disconnected data Suppliers, inventory, orders, and revenue never linked Interactive Knowledge Graph visualizing all business relationships
No expert advice on demand Expensive consultants; no 24/7 advisory AI Copilot powered by Google Gemini for real-time advice
Poor reporting & export No structured business reports for stakeholders PDF, CSV, and Excel export with professional formatting

1.3 Target Audience

  • SME Owners & Founders — Need high-level strategic overview and decision simulations
  • Operations Managers — Need inventory, supplier, and fulfillment insights
  • Finance Teams — Need scenario comparisons, ROI analysis, and exportable reports
  • Business Analysts — Need analytics dashboards, knowledge graphs, and data exports

2. Project Overview & Objectives

2.1 What is DecisionPilot AI?

DecisionPilot AI is a full-stack, AI-powered business decision intelligence platform built for SMEs. It transforms raw business variables — revenue, operational costs, inventory levels, supplier reliability, and market conversion rates — into actionable, quantified strategic intelligence using Monte Carlo simulation, Google Gemini AI, and interactive data visualization.

Core Mission: Enable any SME operator to make data-backed decisions in minutes — not weeks — without needing a data science team or management consultant.

2.2 Key Objectives

  1. Provide quantified scenario simulation with statistically validated outcomes using Monte Carlo modeling
  2. Deliver AI-generated consulting advice (via Google Gemini 1.5 Flash) for each simulated decision option
  3. Visualize business entity relationships through an interactive Knowledge Graph
  4. Monitor inventory health and supplier reliability in real time
  5. Enable professional report generation in PDF, CSV, and Excel formats
  6. Maintain enterprise-grade security via Supabase JWT authentication with role-based access control
  7. Deploy as a cloud-native, fully managed SaaS application

2.3 Project Scope

The platform covers the following high-level operational domains for SMEs:

  • Strategic Decision Intelligence (Simulate → Compare → Act)
  • Financial Analytics (Revenue, Cost, Profit, ROI tracking)
  • Supply Chain Intelligence (Inventory + Supplier management)
  • Business Knowledge Mapping (Knowledge Graph)
  • AI Advisory (Copilot chat assistant)
  • Compliance Reporting (Exportable business reports)

3. System Architecture

3.1 High-Level Architecture

DecisionPilot AI follows a modern 3-tier architecture with clear separation between the presentation layer, business logic layer, and data layer.

🖥️ PRESENTATION LAYER — React + TypeScript (Vercel)

Single Page Application built with React 18, TypeScript, Vite, TailwindCSS, Framer Motion. Handles all UI rendering, routing, state management, and user interactions. Deployed on Vercel with automatic CI/CD from GitHub.

⚙️ BUSINESS LOGIC LAYER — FastAPI + Python (Render)

RESTful API server built with FastAPI. Handles Monte Carlo simulation (NumPy), Gemini AI integration, JWT verification, rate limiting, and all business domain logic. Deployed on Render.com with auto-deploy from GitHub.

🗄️ DATA LAYER — Supabase (PostgreSQL)

Managed PostgreSQL database provided by Supabase. Handles user authentication, session management, JWT token issuance, and persistent workspace data storage. Also provides Row Level Security for multi-tenant isolation.

🤖 AI LAYER — Google Gemini 1.5 Flash

External AI API called by the FastAPI backend. Generates structured JSON consulting reports including supporting factors, risks, opportunities, reasoning, and alternative strategies for each simulated decision option.

5.5 Simulate Decision (New Decision)

Route: /decisions/new  |  Access: Owner and Admin roles only

Purpose

The primary intelligence engine. Allows business operators to configure real business parameters, define competing strategy options, run a Monte Carlo simulation, and receive AI-generated consulting advice — all within a single guided workflow.

Key Features

Step 1 — Configure Variables

  • Input baseline financial variables: Monthly Revenue, Operating Costs (Opex), Conversion Rate
  • Variables are validated with Pydantic schemas before submission
  • Dynamic form with add/remove variable capability

Step 2 — Define Options

  • Create multiple competing strategy options (e.g., "Hire Sales Reps", "Cut Opex by 15%")
  • Each option includes: name, opex modification %, conversion rate lift %
  • Minimum 2 options required for meaningful comparison

Step 3 — Monte Carlo Simulation (Backend)

  • Calls POST /api/v1/decisions/{id}/simulate
  • Backend SimulationEngine runs 1,000 trials per option using NumPy normal distributions
  • Simulates revenue, cost, and profit variance across all trials
  • Generates 4-checkpoint trend trajectories (Month 1 to Month 4)
  • Computes ROI for each option: ROI = ((mean_revenue - mean_cost) / mean_cost) × 100
  • Performs Sensitivity Analysis: identifies which variables drive the most variance
  • Returns a Confidence Score (93.5% default) for the simulation run

Step 4 — Gemini AI Consulting Report

  • For each simulated option, calls Google Gemini 1.5 Flash via async httpx
  • Structured JSON prompt requesting: supportingFactors, reasoning, risks, opportunities, alternativeStrategy
  • AI fallback: If Gemini API fails, a high-quality mock consulting report is returned
  • Results stored in backend session memory and returned to frontend

Technology Used

NumPy (Monte Carlo) Google Gemini 1.5 Flash FastAPI Pydantic Framer Motion (stepper UI)

7. Security & Authentication

7.1 Authentication Flow

  1. User logs in via Supabase Auth (email + password)
  2. Supabase validates credentials and issues a signed JWT (HS256 or RS256)
  3. React frontend stores the JWT in Zustand authStore (memory) — never in localStorage
  4. All API requests include Authorization: Bearer {token} header injected by ApiClient
  5. FastAPI's HTTPBearer security dependency extracts the token
  6. get_current_user() verifies the JWT signature using SUPABASE_JWT_SECRET (HS256) or JWKS endpoint (RS256)
  7. JWT claims returned as user context to all route handlers

7.2 CORS Policy

FastAPI uses the native CORSMiddleware (pure ASGI) configured with:

  • Explicit origins: http://localhost:5173, https://decision-pilot-ai-alpha.vercel.app
  • Regex pattern: https://[a-zA-Z0-9][a-zA-Z0-9\-]*\.vercel\.app — covers all Vercel preview deployments
  • Allowed methods: GET, POST, PUT, DELETE, OPTIONS, PATCH
  • Credentials: Enabled

7.4 Data Security

  • All API communication is over HTTPS (TLS/SSL)
  • JWT tokens are short-lived and auto-refreshed by Supabase SDK
  • Passwords are hashed using bcrypt (passlib) — never stored in plaintext
  • Supabase enforces Row Level Security (RLS) on all database tables
  • API keys (Gemini, Supabase) stored as environment variables — never in source code
  • Report exports encrypted with Encryption Enabled label (Supabase datastore sync)
